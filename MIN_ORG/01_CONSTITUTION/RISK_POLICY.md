# RISK_POLICY

목적: 승인 병목 없이도 안전성을 유지하도록 리스크 기반 승인 체계를 고정한다.

## 1) 리스크 레벨
- L0: 조회/읽기/시뮬레이션 (무부작용)
- L1: 로컬 로그/파일 생성 (경미한 부작용)
- L2: 금전/주문/포지션/외부 자원 변경
- L3: 비가역 작업(삭제/권한변경/키 회전/배포)

## 2) 승인 규칙
- L0: 승인 없이 가능
- L1: 정책 기반 자동 승인 가능
- L2: 사람 승인 필수
- L3: 사람 승인 + 2단계 확인(쿨다운) 필수

## 3) 공통 가드레일
- request_hash 일치 검증
- approval_id 연결 검증
- budget_cap / risk_limit 초과 시 즉시 중단
- kill switch 우선
- evidence_id 없이 종료 금지

## 4) 중단(Stop) 트리거
- 손실 한도 초과
- 승인 불일치
- 이상 징후(오류율 급등, 로그 누락)
- 정책 위반 감지

## 5) 사고 처리
1. 즉시 중단
2. 안전 상태 복귀(가능 시 롤백)
3. incident 기록
4. 재발 방지 룰 업데이트

## 6) 원칙
- 속도보다 안전
- 자동화보다 통제
- 통제보다 증거
